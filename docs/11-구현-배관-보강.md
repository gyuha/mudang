설계 검토에서 드러난 **구현 직전 배관 갭**을 닫는다. 전부 결정 갈래가 아니라 명세 — 추천 스펙으로 박는다. (위협 모델 #1·서사 전달 #6의 *결정*은 [00] D6-a / 아래 D23으로 확정됨)

## 1. 동료 공격 · 투사체 시스템 (P0 #2)

[06]의 `EnemySystem`은 *적*만 배열로 관리한다. 동료(소수 노드)가 그 배열을 **어떻게 때리는가**가 비어 있었다.

원칙: 동료는 ≤4명이라 일반 노드. 공격은 **적 spatial hash 질의 → `EnemySystem.apply_damage(idx, amount)`**로 처리. 적은 노드가 아니므로 동료도 적 노드를 참조하지 않고 **인덱스/위치로 타격**한다.

| 역할 | 공격 방식 | 구현 |
|------|----------|------|
| 화랑(근접) | 즉시 판정(투사체 없음) | 공격 주기마다 `engage_range` 부채꼴 내 적 질의 → 데미지. 히트스캔식 |
| 활잡이(원거리) | **투사체 풀** | 발사 → 직선 이동 → 매 프레임 진행 경로 spatial hash 질의 → 명중 시 데미지, 관통 카운트 소진까지 유지 |
| 견습무당(힐) | 대상 직접 | 주기마다 최저HP 아군 회복(투사체 없음) |

```gdscript
# EnemySystem 공개 API (동료/주술이 호출)
func query_circle(center: Vector2, r: float) -> PackedInt32Array   # 적 인덱스들
func apply_damage(idx: int, amount: float) -> void                  # 0 이하 → 처치+드랍
func position_of(idx: int) -> Vector2
```

투사체(`CompanionProjectile`): 화살은 동시 수십 개라 노드 풀로 충분(적처럼 수백이 아님). 풀링 필수, Area2D 대신 spatial hash 질의로 명중 판정([06] 원칙 일관).

> 무녀는 공격이 없으므로 이 시스템을 쓰지 않는다. 「물렀거라」는 데미지 0(변위만), 오라는 감속만.

## 2. 게임 상태 · 일시정지 · 씬 전환 (P0 #3)

`Main`이 화면을 교체한다고만 했고 *상태 기계·일시정지 구현*이 없었다 → M0 블로커.

### 2.1 GameState (autoload)
```gdscript
# game_state.gd (autoload)
enum S { BOOT, DASHBOARD, BRIEFING, LOADOUT, RUN, LEVELUP_PAUSE, RESULT }
var state: S
signal state_changed(new_state)
```
화면 전이([10]§1)는 이 상태를 따라 `Main`이 현재 화면 노드를 교체(또는 `change_scene_to_packed`). 대시보드/브리핑/편성/설정/결과는 `Control` 화면.

### 2.2 일시정지 모델
| 상황 | 방식 |
|------|------|
| 무녀 레벨업 3택 | `get_tree().paused = true`. 3택 UI는 `process_mode = ALWAYS`. 런 시스템은 기본 `PAUSABLE`이라 멈춤 ([03]§2 D12) |
| 동료 비정지 카드 | **멈추지 않음**(D12). 선택 UI만 표시. 선택 순간 약한 슬로모는 `Engine.time_scale` 옵션(노브) |
| ESC 일시정지 메뉴 | `paused=true` + 메뉴(재개/설정/포기) |
| 대시보드/브리핑 | 런이 아니므로 일시정지 개념 없음 |

> 핵심: 무녀 3택만 진짜 멈춤, 동료 3택은 흐름 유지 — D12 그대로를 `process_mode`로 구현.

## 3. 세이브/로드 스키마 + 메타 업그레이드 (P1 #4)

[06]은 `user://save.json` 언급뿐, [03]§5 "소폭 영구강화"가 추상적이었다.

### 3.1 세이브 파일
- `user://save.json` — MetaProgress(진행). `user://settings.cfg` — 설정(별개, [10]§5).
- **버전 필드 + 마이그레이션 훅** 필수(스키마 진화 대비).

```json
{
  "version": 1,
  "unlocked_stages": ["stage_hwalinseo"],
  "unlocked_companions": ["seubseup_mudang"],
  "loadout_slots": 2,
  "meta_currency": 0,
  "meta_upgrades": { "mudang_hp": 0, "loadout_slot": 0 },
  "stage_records": { "stage_hwalinseo": { "cleared": false, "best_time": 0 } },
  "tutorial_done": false
}
```
```gdscript
# meta_progress.gd 로드 시
if data.version < CURRENT_VERSION: data = _migrate(data)
```

### 3.2 메타 업그레이드 목록 (가볍게 — D15 상한)
| id | 효과 | max | 비고 |
|----|------|-----|------|
| `mudang_hp` | 무녀 시작 HP +10 | 5 | |
| `loadout_slot` | 출전 슬롯 +1 | 2 | 2→4 (D16) |
| `companion_start_lv` | 동료 시작 레벨 +1 | 3 | |
| `revive_speed` | 기본 부활 속도 +5% | 3 | |
| `start_soulfire` | 시작 시 무녀 혼불 약간 보유 | 3 | 초반 레벨 가속 |

> 전부 **소폭**. 메타가 세지면 스테이지 난이도 설계가 메타 수치에 종속돼 깨진다(D15 거부 사유). 재화: 런 정산의 `원혼 정수`.

## 4. 튜토리얼 · 온보딩 · 첫 실행 (P1 #5)

[10]은 1장을 튜토리얼이라 했으나 *학습 흐름*이 없었다.

### 4.1 첫 실행 플로우
```
타이틀 → (세이브 없음 → tutorial_done=false) → 1장 강제 진입(튜토리얼 모드) → 클리어 시 tutorial_done=true → 대시보드
       → (세이브 있음) → 대시보드
```

### 4.2 1장 튜토리얼 단계 (트리거 기반, 단계별 프롬프트)
| 순서 | 학습 | 트리거 |
|------|------|--------|
| 1 | 이동 | 게임 시작 → "WASD/조이스틱으로 움직여라" |
| 2 | 오라 감속 | 첫 적 접근 → "주변 적이 느려진다(오라)" |
| 3 | 「물렀거라」 | 적이 동료에 근접 → "탭해서 밀어내라" |
| 4 | 혼불 줍기/전달 | 첫 혼불 드랍 → "무녀 혼불=성장, 동료 혼불은 동료에 다가가 전달" |
| 5 | 쓰러짐/부활 | 동료 첫 쓰러짐 → "곁에서 부활시켜라" (미니맵 알림 강조, [08]§6) |
| 6 | 모여라 | 위기 상황 → "모여라로 재정비" |

데이터: `TutorialStep`(메시지·트리거 조건·강조 대상). 1회성(`tutorial_done`). 강조는 해당 UI/대상 하이라이트 + 약한 슬로모.

## 5. 서사 전달 — 미니멀 텍스트 카드 (P1 #6 / D23)

StageDef 데이터에 서사 카드를 얹는다(저작 도구로 추가 가능). 게임플레이 우선, 서사는 경량.

```gdscript
# StageDef 추가 필드(예약 → 구현 시)
@export var intro_cards: Array[NarrativeCard] = []   # 장 시작
@export var outro_cards: Array[NarrativeCard] = []   # 장 끝(클리어)

class_name NarrativeCard extends Resource
@export var illustration: Texture2D       # 정적 일러스트(옵션)
@export var speaker: String               # 화자(옵션)
@export_multiline var text: String
```
- 장 시작/끝에 카드 시퀀스 재생(탭으로 넘김, **스킵 가능**).
- 브리핑의 "서사 한 줄"([10]§3)과 별개 — 브리핑은 요약, 카드는 연출.
- 텍스트는 i18n 키로(§6.2).

## 6. P2 — 후속으로 명시 (지금 구현 안 함, 구조만 인지)

### 6.1 엔딩 분기 (A/B/C)
6장 클리어 후 선택 → `ending_flag` 저장. 분기 연출은 텍스트 카드 재사용. 6장 콘텐츠와 함께 후속.

### 6.2 i18n
모든 표시 텍스트를 **키-테이블**(csv/`.po`)로. 한국어 기본. **규칙: 표시 문자열 하드코딩 금지**(처음부터 키로) — 나중 다국어가 재작업이 안 되게. 구조만 1일차에 예약, 번역은 후속.

### 6.3 오디오 시스템
버스: `Master / BGM / SFX / Voice`. `AudioStreamPlayer` 풀(동시 SFX 상한), 거리 기반 볼륨, BGM 페이즈 크로스페이드([08]§4). 설정 볼륨([10]§5)이 버스 볼륨에 매핑.

### 6.4 테스트 전략 (CLAUDE.md 검증 원칙 반영)
| 대상 | 방법 |
|------|------|
| 성능 | 500마리 벤치 씬 — 60FPS 자동 측정(M1 게이트, [07]) |
| 데이터 | `StageDef.validate()` + 전체 `data/**.tres` 로드 스모크 테스트(누락/깨진 참조 검출) |
| 핵심 로직 | spatial hash 질의, 넉백 변위, 혼불 전달/회복 계산 → GUT 단위 테스트 |
| AI 거동 | 수동 플레이테스트(자동화 과투자 금지, YAGNI) |

> 테스트는 **핵심 수치·검증 로직**에 집중. AI·연출은 플레이테스트로. 프레임워크 과설치 금지.
