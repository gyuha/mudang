# 06. 기술 아키텍처

엔진 Godot 4 / GDScript(D1), 2D 탑다운(D2), PC 우선·모바일 이식(D3), 동시 적 ~500(D19).

핵심 제약: **500마리 환경에서 "1마리 = 1 Node2D + Area2D + 물리바디"는 불가.** 처음부터 데이터 지향으로 간다.

---

## 1. 적 표현 — Node 금지, 데이터 배열 + MultiMesh (D19)

적은 노드가 아니라 **구조화 배열(SoA 권장)**로 관리하고, **MultiMeshInstance2D**로 한 번에 그린다.

```
EnemySystem (Node)
  arrays: position[], velocity[], hp[], def_idx[], state[]  (PackedVector2Array 등)
  - update(dt): 이동/AI/넉백 적용 (벡터화 루프)
  - render: MultiMesh.set_instance_transform_2d(i, ...) 일괄
  - 풀: 죽으면 swap-remove로 빈 슬롯 재사용(할당 0)
```

- 그리기: `MultiMeshInstance2D` + 적 종류별 멀티메시(또는 텍스처 아틀라스 + 인스턴스 커스텀데이터로 프레임 선택).
- 애니메이션: 셰이더에서 `INSTANCE_CUSTOM`으로 스프라이트 프레임/색조 제어(노드 AnimatedSprite 500개 금지).
- 동료/무녀/보스처럼 **수가 적고 개성 강한 것만** 일반 노드(AnimatedSprite2D)로.

> 처음부터 이 구조로 짤 것. "일단 노드로 만들고 나중에 최적화"는 500마리에서 전면 재작성이 된다(되돌리기 비용 큼 = D19를 ADR급으로 취급).

---

## 2. 충돌/근접 질의 — Spatial Hash (물리 없음)

물리 엔진/Area2D를 적에 쓰지 않는다. 대신 **균일 격자 spatial hash**로 근접 질의.

```
SpatialHash
  cell_size ≈ 64 px
  insert(id, pos), query_circle(center, r) -> ids
용도:
  - 오라 감속(무녀 주변 적)
  - 넉백(클릭 위치 주변 적)
  - 혼불 자석(무녀 주변 혼불)
  - 동료 타게팅(주변 적), separation(주변 동료)
  - 적 contact damage(동료/대상 주변 적)
```

매 프레임 재구축(클리어+재삽입) 또는 이동량 기반 갱신. 500마리면 재구축이 더 단순하고 충분.

## 3. 씬 트리 구조

```
Main (씬 전환/메타 진입점)
├─ MetaProgress (autoload)
├─ InputAdapter (autoload 또는 노드)
└─ RunScene (런 1회)
   ├─ Camera2D (무녀 추적)
   ├─ Background (StageDef.background)
   ├─ Mudang (AnimatedSprite2D + 레버 로직)
   ├─ CompanionManager → Companion[] (노드, 소수)
   ├─ EnemySystem (배열 + MultiMesh)
   ├─ SoulfireSystem (혼불 배열 + MultiMesh)
   ├─ WaveDirector (타임라인+예산, StageDef 주입)
   ├─ ObjectiveManager (목표 모듈)
   ├─ SpatialHash (적/동료/혼불 별도 인스턴스)
   ├─ HUD (레벨/쿨/쓰러짐 알림/3택 UI)
   └─ MinimapHUD (Control + _draw, 액터 마커 + 쓰러짐 알림 [08]§6)
```

---

## 4. 업데이트 순서 (한 프레임)

```
1. InputAdapter 갱신
2. SpatialHash 재구축(적/혼불/동료)
3. Mudang: 이동, 넉백(입력 시), 오라 적용, 혼불 전달, 모여라, 부활 채널
4. CompanionManager: 각 동료 FSM 틱(0.1~0.2s 주기) + 이동/공격
5. EnemySystem: AI 이동(타겟 방향+separation+오라 감속) + contact damage + 넉백 변위 소거
6. WaveDirector: 예산/타임라인 스폰
7. SoulfireSystem: 자석/픽업/전달
8. ObjectiveManager: 승패 판정
9. 렌더: MultiMesh transforms 일괄 업데이트
```
